Seamless in-page editing with ngContentEditable...

  • Simple

    Why can't the user just edit stuff, right there, on the page?

    Why does the developer have to work with intrusive components which interfere with page style, layout or functionality?

    The answer to both these questions is ngContentEditable!

  • Fun

    The idea was to create an extremely unobtrusive directive for AngularJS developers which makes any element within your application editable.

    This type of thing has been done before, to varying degrees, but none specifically designed to stay the hell out of your way while at the same time providing a stupidly simple interface for any user.

  • Seamless

    All you have to do is include the ngContentEditable directive anywhere in your view. Hook up your data model (required) and away you go!.

    You can register your own directives and implement specific behavior when they are contained within an editable region.

  • Simply drop into your page
  • Implement your own UI
  • Supports native drag and drop
  • Most modern browsers supported

About ngContentEditable

Browser support

This directive will work in most modern browsers that support contenteditable attribute for elements and native drag and drop.

As usual, browsers implement these features differently and in many cases, deviate wildly in terms of API for range selection and what not. So, beware...

The idea for this directive is proof-of-concept, so hang tough for future revisions with improved cross browser feature compatability.

Use of the Foxy browser is strongly recommended. Chrome is good (enough). Don't try this in IE8, kids. ;-)

Howto...

Instructions

It's easy... Just load the module into your application. Then hook up your ng-model and "editable" class name for elements within your view.

Now, for the slightly more involving part...

You'll need to implement your own user interface. For this, it's recommended to use the readily available directive for converting any element within your view to a ngContentEditable command. Just add the "editable-control" class name to these elements. Alternatively, of course, you could totally roll your own.

Finally, you may want to support native drag-and-drop on your editable regions. Just add the "editable-drop" class name to those elements.

You can do lots of other imaginative stuff, but for now, just check out the example application to see a possible usage scenario and how it's implemented.
